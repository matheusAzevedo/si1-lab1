1- Desenvolver Sistemas Web:

Um dos pontos de maior foco da Disciplina é o desenvolvimento de sistemas Web, modelo esse, entrando cada vez mais em ascenção, garantindo a base e trazendo inúmeros benefícios para empresas nos mais diversos ramos de atuação. Além de mais rentáveis e econômicas, as aplicações Web oferecem uma melhor estabilidade. A integridade e segurança dos dados é garantida por acesso restrito apenas a usuários autorizados, sendo essas informações, atualizadas em tempo real, garantindo assim a comodidade e praticidade em meio ao funcionamento de uma corporação. Logo, uma boa opção para alguém que deseja se firmar na área de TI.

2- Familiarizar-se com ferramentas de controle de versão:

Um programa de controle de versão é um software que age sob o objetivo de gerenciar diferentes versões no desenvolvimento de um aplicativo qualquer. Seu conceito está relacionado ao desenvolvimento de softwares livres, comunente em empresas e instituições de tecnologia e visa sobretudo, controle de histórico, ou seja, facilidade no resgate de versões mais antigas e estáveis; Ramificação de projeto, possibilitando a divisão em várias linhas de desenvolvimento, podendo ser trabalhadas paralelamente, sem que uma interfira na outra; Além do trabalho em equipe, permitindo assim, que diversas pessoas trabalhem simultaneamente, minimizando o tempo e o desgaste provocado por problemas com conflitos de edições.

3- Compreender diferentes tipos de Padrão de Projeto:

Padrões de Projeto são uma solução geral reutilizável para um problema que ocorre com frequência dentro de um determinado contexto no projeto de software. De forma direta, são um uma descrição ou modelo de como resolver um problema que pode ser usado em muitas situações diferentes. Quando usadas adequadamente, constitui o melhor tipo de prática que um programador pode usar para resolver problemas comuns quando projetar uma aplicação ou sistema, visando o reusode soluções pré existentes.

4- Lidar com técnicas de TDD:

TDD (Em inglês "Test Drive Development") é uma técnica de desenvolvimento de software que baseia em um ciclo curto de repetições: Primeiramente o desenvolvedor escreve um caso de teste automatizado que define uma melhoria desejada ou uma nova funcionalidade. Então, é produzido código que possa ser validado pelo teste para posteriormente o código ser refatorado para um código sob padrões aceitáveis. Um estudo de 2005 descobriu que usar TDD significava escrever mais testes, e logo, programadores que escreviam mais testes tendiam a ser mais produtivos além de oferecer mais do que somente um maneira simples de validação e de correção, podendo também, orientar o design de um programa (ponto crítico da Disciplina).

5- Ter noções de Arquitetura de Software:

A arquitetura de software de um sistema consiste na definição dos componentes de software, suas propriedades externas, e seus relacionamentos com outros softwares. O termo também se refere à documentação da arquitetura de software do sistema. A documentação da arquitetura do software facilita: a comunicação entre os stakeholders, registra as decisões iniciais acerca do projeto de alto-nível, e permite o reúso do projeto dos componentes e padrões entre projetos. Num software, um projeto arquiteturial traz desempenho, portabilidade, confiabilidade, disponibilidade. Numa empresa,o arquiteto de Software, papel de suma importância numa estratégia, precisando para isso, ter profundo conhecimento do domínio, das tecnologias existentes e de processos de desenvolvimento de software.
